# Security Policy

## Reporting Issues

Report security-sensitive issues privately to the repository maintainers. Do not open a public issue containing confidential filings, non-public company information, personal data, credentials, or proprietary analysis.

## Data Handling

This repository does not require credentials or network access. Users may supply public-company documents for review, but they remain responsible for handling confidential or restricted information appropriately.

Do not commit:

- Non-public company documents.
- User-provided confidential materials.
- Local analysis notes.
- API keys or credentials.
- Environment files.
- Leak-term files used by compliance scripts.

## Supported Use

The skill is intended for document analysis and question generation. It should not be used to harass, defame, manipulate markets, evade professional obligations, or publish unsupported allegations.
