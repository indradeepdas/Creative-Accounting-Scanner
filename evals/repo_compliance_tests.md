# Repo Compliance Tests

Before release, verify:

1. All required files exist.
2. Every scanner contains the required sections.
3. No non-public build inputs are referenced.
4. No local notes or supplied documents are inside the repository.
5. The README describes the project without provenance language.
6. The disclaimer is legally careful.
7. Contribution guidance prohibits copied material and unsupported accusations.
8. The private leak-term scanner passes when supplied with the maintainer's leak-term list.
9. The accusatory-language scanner passes.
10. CI runs all repository compliance scripts.
